HTML

  • The head element contains information about the webpage and can improve crawlability by search engines
  • The body element represents the visible content shown to the user
  • Semantic HTML calls for using as many semantically tags/elements as possible to make code more readable and accessible
  • H1 tags should only be called once on the page

CSS

  • For simplifying margin or padding CSS, you can start with 'top-bottom' pixels, and then do 'left-right' pixels, so as to only have two numbers
  • padding makes an element larger, and exists *inside* the element
  • margin adds space to the *outside* of the element
  • display: block; indicates that each item should occupy a new line

Git

  • Git is one of the primary ways that developers handle the problem of version control
  • Git flow is the process of iteratively updating and mantaining code in GitHub as issues are worked through and features are added
  • SSH keys can be attached to personal devices and uploaded to GitHub to automatically connect to GitHub without having to log in every time
  • Feature branches should be created before beginning work on the code in your repository so as not to mess up the main branch
